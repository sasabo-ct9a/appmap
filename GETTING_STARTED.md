# AppMap 導入マニュアル(macOS 版)

AppMap は、あなたが(または AI が)作ったアプリを **1 枚の地図** に変換するデスクトップアプリです。このマニュアルでは、初めて使うところまでをご案内します。

**対応 OS**: macOS 13 Ventura 以降(Apple Silicon / Intel どちらも可)
**所要時間**: 約 15〜20 分(Node.js の初回インストールを含む)

---

## 必要なもの

- macOS 13 Ventura 以降
- インターネット接続
- **Claude Pro または Max プラン**(月 $20 〜) — 自分で契約しておく必要があります(<https://claude.com/>)

> AppMap はあなたの代わりに **Claude(AI)** にコードを読んでもらい、その結果を地図にします。Claude を呼び出す権利が必要なので、Claude のプラン契約が前提です。

---

## ステップ 1: Node.js をインストール

AppMap が裏で使うプログラムを動かす土台です。Bubble でいうと「Bubble そのもの」のような基盤にあたります。

### 手順

1. <https://nodejs.org/> を開く
2. 「**LTS**」と書かれた **緑のボタン** をクリック(2026 年現在 v22.x)
3. ダウンロードされた `.pkg` をダブルクリック
4. インストーラを **「続ける → 続ける → インストール」** で進める(設定は全部デフォルトで OK、途中で Mac のパスワードを求められたら入力)
5. 完了

### 確認

**ターミナル** を開きます:

- Spotlight 検索(`⌘ + Space`)→「**ターミナル**」と入力 → Enter
- または:Finder → アプリケーション → ユーティリティ → ターミナル

開いたターミナルに、以下をコピペして Enter:

```
node --version
```

`v22.xxx` のような数字が出れば成功です。

---

## ステップ 2: Claude Code CLI をインストール

AppMap と Claude を繋ぐプログラムです。

ターミナルで:

```
npm install -g @anthropic-ai/claude-code
```

しばらく文字が流れて、最後に `+ @anthropic-ai/claude-code@2.x.x` のような行が出れば成功です(`npm warn deprecated ...` は無視して OK)。

### `EACCES` という権限エラーが出たら

Mac の権限の都合で、管理者として実行する必要があります:

```
sudo npm install -g @anthropic-ai/claude-code
```

Mac のログインパスワードを求められたら入力(画面には表示されません、入力して Enter で OK)。

### 確認

```
claude --version
```

`2.1.xxx (Claude Code)` のような出力が出れば OK。

---

## ステップ 3: Claude にログイン

```
claude login
```

ブラウザが自動で開き、Anthropic のログイン画面が出ます。Claude Pro/Max のアカウントでログインしてください。

完了すると、ターミナル側に「**Successfully authenticated**」が表示されます。

## ステップ 4: AppMap を起動

[配布形式に合わせて確認してください]

- **`.dmg` 版**: ダブルクリックでマウント → 開いたウィンドウで `AppMap.app` を **アプリケーション** フォルダにドラッグ → アプリケーションから AppMap を起動
- **`.app.zip` 版**: ダウンロードファイルをダブルクリックで解凍 → `AppMap.app` を **アプリケーション** フォルダにドラッグ → 起動

### 初回起動時 — Gatekeeper の警告への対処

AppMap は個人開発のため Apple の公式署名がありません。そのため、macOS が安全のために起動を止めることがあります。**安全な配布元(私)からのファイルなので、以下の手順で許可してください**。

#### パターン A:「開発元が未確認」の警告

「**"AppMap" は、開発元が未確認のため開けません**」と出た場合:

1. アプリケーションフォルダで `AppMap.app` を **右クリック**(またはトラックパッドで二本指タップ / `Control + クリック`)
2. メニューから「**開く**」を選択
3. ダイアログで再度「**開く**」をクリック

これで起動でき、以降は普通にダブルクリックで開けます。

#### パターン B:「このまま開く」を使う(Sequoia 以降に多い)

パターン A で開かない、または「**このまま開く**」ボタンが出る場合:

1. アップルメニュー  → **システム設定**
2. **プライバシーとセキュリティ** を選択
3. 下にスクロール →「**"AppMap" は開発元を確認できないためブロックされました**」の右にある「**このまま開く**」をクリック
4. もう一度 AppMap を開く

#### パターン C:「壊れているため開けません」と出る場合

quarantine 属性が深く付いている状態です。ターミナルで以下を実行:

```
xattr -d com.apple.quarantine /Applications/AppMap.app
```

その後、改めて AppMap を起動してください。

## ステップ 5: 最初の使い方

### 5-1. 起動直後 — サンプルマップを見る

最初に **サンプルマップ**(AppMap 自身の画面構造)が表示されます。

- **上段「主フロー」**: メインで使う画面
- **下段「サブ画面」**: 詳細パネル・設定など、メインから開く画面
- **四角(ノード)** = 1 画面、**矢印** = 画面間の移動
- **破線の矢印** = 階層をまたぐ移動(メインからサブを開く 等)

> サンプルは「AppMap が自分自身を地図にしたら、こうなるよ」というデモです。

### 5-2. ノードをクリック → 詳細を見る

四角を 1 つクリックすると、右側に **詳細パネル** が開きます:

- その画面が何をするかの説明
- **関連ファイル** — ソースコード上どこに対応するか
- **関連ノード(つながる画面)** — どの画面から / どの画面へ移動するか

閉じるときは右上の **×**。

### 5-3. ノーコード語トグル(画面右上)

説明文に出てくる技術用語が苦手な方は、右上の **「ノーコード語」** トグルを ON にしてください。

- 「Description」→「概要」
- 「関連ノード」→「つながる画面」
- 「主フロー / サブ画面」→「主な画面 / 開いて出る画面」
- 本文中の専門用語が、Bubble / Notion / Glide の言葉に翻訳されます

説明文に **点線の下線** が引かれた語があれば、その上にマウスをホバーすると **その用語の解説** が出ます(例:API、データベース、認証 など)。

### 5-4. 自分のアプリを分析する

1. 左上の **「フォルダを選ぶ」** をクリック
2. **自分のアプリのソースコードのフォルダ** を選択
   - 例:AI ツール(Cursor / Claude Code / v0 等)が出力したフォルダ
   - 例:Bubble のエクスポート結果のフォルダ
3. 「**分析中…**」と表示され、経過秒数がカウントされる
4. **数十秒〜数分** 待つ(フォルダのサイズによる)
5. 完了するとマップが新しい内容に切り替わる
6. 画面上部に **コスト($)** と **ノード数 / エッジ数** が表示される

### 5-5. 履歴で過去のマップを切り替える

「**履歴 (N) ▾**」ボタンをクリック → 過去に分析したフォルダ一覧。

- 行をクリックで切替
- **×** で履歴から削除
- アプリを終了 → 次回起動時、最後に見ていたマップが自動で復元されます

### 5-6. サンプルに戻す

「**サンプルに戻す**」ボタンで、いつでも最初のサンプルマップに戻れます(履歴は消えません)。

## 困ったとき

| 症状 | 対処 |
|---|---|
| AppMap 起動時「Claude Code CLI が見つかりません」 | ステップ 2 が完了しているか確認。ターミナルを **一度閉じてもう一度開いて** から `claude --version` を試す |
| `npm install -g` で `EACCES` エラー | `sudo npm install -g @anthropic-ai/claude-code` で再実行(Mac のパスワード入力) |
| AppMap 起動時「Claude に認証されていません」 | ターミナルで `claude login` をもう一度実行 |
| 「分析中」のまま 5 分以上動かない | 大規模なフォルダだと時間がかかります。10 分超えるなら AppMap を一度終了 → 起動 |
| 「AppMap.app は壊れているため開けません」 | ターミナルで `xattr -d com.apple.quarantine /Applications/AppMap.app` を実行してから再起動 |
| 「開発元を確認できない」とブロックされた | システム設定 → プライバシーとセキュリティ → 下のほう「このまま開く」をクリック |
| マップが出たけど内容がしっくりこない | AI の解釈ブレです。「サンプルに戻す」→「フォルダを選ぶ」で再分析するか、別フォルダで試す |
| エラー本文が表示された | 本文をマウスで選択してコピーできます。私(AppMap 作者)に送ってもらえると改善材料になります |

## このアプリで覚えておいてほしい 3 つの原則

1. **全体を先に見せる、詳細は後で開く**
   最初はマップ全体だけ。気になるノードを **クリックして初めて** 詳細が開きます。最初から全部見せない設計です。

2. **専門用語は翻訳できる**
   「ノーコード語」トグルで、技術用語を Bubble/Notion/Glide の言葉に翻訳します。説明文の点線部分にホバーすると、用語ごとの解説も出ます。

3. **AI は完璧じゃない**
   AppMap は Claude という AI に、あなたのコードを読ませて地図化しています。AI が画面を見落としたり、変な分け方をすることがあります。気付いたら「サンプルに戻す」→ 再分析、または別の角度から見てください。
